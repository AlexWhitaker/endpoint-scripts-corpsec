# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report security issues by emailing the maintainer directly. You can find contact details on the GitHub profile. Include a description of the issue, reproduction steps, and any suggested fix if you have one. Expect a response within 5 business days.

## Credentials

None of these scripts contain hardcoded credentials. Sensitive values (e.g., Tenable linking key) are passed via environment variables or MDM script parameters. If you find a hardcoded secret, please report it immediately.
